Create `diskcomp/ansi_codes.py` with ANSI escape code constants and helper functions. Implement:

1. **Color constants** (standard 16-color ANSI):
   - `CYAN = "\033[36m"` (for scanning indicator)
   - `GREEN = "\033[32m"` (for completion tick)
   - `RED = "\033[31m"` (for errors)
   - `RESET = "\033[0m"` (reset formatting)
   - `BOLD = "\033[1m"` (for banners)

2. **Symbol constants**:
   - `ARROW = "→"` (scanning indicator)
   - `TICK = "✓"` (completion indicator)
   - `CROSS = "✗"` (error indicator)

3. **Helper functions**:
   - `colored(text: str, color: str) -> str`: Wraps text with color code and reset. Example: `colored("Scanning...", CYAN)` returns `"\033[36mScanning...\033[0m"`
   - `progress_bar(current: int, total: int, width: int = 40) -> str`: Renders a simple ANSI progress bar. Example: `progress_bar(50, 100)` returns `"[████████░░░░░░░░░░░░░░░░░░░░░░░░] 50%"` with filled/empty block chars. Include percentage at the end.
   - `format_speed(bytes_per_sec: float) -> str`: Converts speed in bytes/sec to human-readable format. Example: `format_speed(1048576)` returns `"1.0 MB/s"`. Handle KB/s, MB/s, GB/s with 1 decimal place.
   - `format_eta(seconds: float) -> str`: Converts remaining seconds to human-readable format. Example: `format_eta(125)` returns `"2m 5s"`. Handle hours, minutes, seconds.

4. **Module docstring** explaining ANSI terminal support and platform compatibility.

Per RESEARCH.md: These constants centralize ANSI codes and prevent magic string duplication. Graceful ANSI fallback will use these for Windows 10+ compatibility.

No external dependencies; stdlib only.

  <files>diskcomp/ui.py</files>

  <action>
Create `diskcomp/ui.py` with UI classes. Implement:

1. **Graceful Rich import**:
   ```python
   try:
       from rich.progress import Progress, BarColumn, TransferSpeedColumn, TimeRemainingColumn, PercentageColumn, TextColumn
       RICH_AVAILABLE = True
   except ImportError:
       RICH_AVAILABLE = False
   ```

2. **UIHandler factory class** (static methods):
   - `create(force_ansi=False) -> Union[RichProgressUI, ANSIProgressUI]`: Returns appropriate UI instance. If `force_ansi=True` or Rich not available, return ANSIProgressUI; else RichProgressUI.
   - `get_available() -> str`: Returns "rich" or "ansi" indicating which is being used.

3. **RichProgressUI class** (when Rich is available):
   - `__init__()`: Initialize with empty progress context and task IDs dict
   - `start_scan(drive_path: str)`: Begin scan, add task with description "[cyan]→ Scanning {drive_path}..." and total=None (indeterminate)
   - `on_folder_done(folder_path: str, file_count: int)`: Update most recent scan task to "[green]✓[/green] {file_count} files from {folder_path}"
   - `start_hash(total_files: int)`: Begin hash progress, add task with rich progress bar columns (BarColumn, PercentageColumn, TransferSpeedColumn, TimeRemainingColumn), total=total_files, description="[cyan]Hashing files..."
   - `on_file_hashed(current: int, total: int, speed_mbps: float, eta_secs: int)`: Update hash task with current progress (advance), update description to show speed + ETA if available
   - `show_summary(duplicates_mb: float, duplicates_count: int, unique_keep_mb: float, unique_keep_count: int, unique_other_mb: float, unique_other_count: int, report_path: str)`: Print a formatted summary using Rich Panel or Table (show all stats in a box)
   - `close()`: Exit context manager for Progress

4. **ANSIProgressUI class** (fallback when Rich unavailable):
   - `__init__()`: Initialize with simple state tracking (no progress context)
   - `start_scan(drive_path: str)`: Print `"{CYAN}{ARROW} Scanning {drive_path}...{RESET}"`
   - `on_folder_done(folder_path: str, file_count: int)`: Print `"{GREEN}{TICK}{RESET} {file_count} files from {folder_path}"`
   - `start_hash(total_files: int)`: Print `"{CYAN}{ARROW} Hashing {total_files} files...{RESET}"`
   - `on_file_hashed(current: int, total: int, speed_mbps: float, eta_secs: int)`: Print a line with ANSI progress bar and speed/ETA. Example: `"[████░░░░░░░░░░░░░░░░░░░░░░░░░░░] 50% | 25/50 files | 2.5 MB/s | ETA 1m 30s"` using helpers from ansi_codes.py
   - `show_summary(...)`: Print a summary box using box-drawing chars (═, ║, ╔, ╗, ╚, ╝, ╞, ╡). Include all stats inside box.
   - `close()`: No-op (no context to clean up)

5. **Common interface**: Both classes have identical method signatures (same parameter names/types) so they're interchangeable.

Per RESEARCH.md: Callback-based architecture keeps business logic (scanner/hasher) independent of presentation (UI). Both classes will be wired to scanner/hasher callbacks in Wave 2.

Methods should use helper functions from ansi_codes.py (colored, progress_bar, format_speed, format_eta).
  </action>
