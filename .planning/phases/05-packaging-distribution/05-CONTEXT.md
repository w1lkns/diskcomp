# Phase 5: Packaging + Distribution - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Make diskcomp installable and shareable in under 60 seconds. Two parallel install paths:
1. `pip install diskcomp` — installs a `diskcomp` CLI entry point
2. `python3 diskcomp.py` — single-file, zero mandatory dependencies

CI validates cross-platform compatibility. README makes it public-facing.
No PyPI publish this phase — machinery only.

</domain>

<decisions>
## Implementation Decisions

### Single-File Bundling (DIST-01)
- **D-01:** Use a `build_single.py` script to auto-generate `diskcomp.py` by concatenating all package modules at release time. The `diskcomp/` package is the canonical source of truth — `diskcomp.py` is a generated artifact, never edited by hand.
- **D-02:** `build_single.py` should be a dev utility at the project root. It reads modules in dependency order and writes a single `diskcomp.py` with a combined stdlib-only import block and all class/function definitions inlined.
- **D-03:** `diskcomp.py` should include a header comment marking it as generated (e.g., `# Auto-generated by build_single.py — do not edit directly`).

### PyPI Package (DIST-02, DIST-03)
- **D-04:** Create `pyproject.toml` with full metadata: name, version, description, author, license, Python requires (`>=3.8`), entry point (`diskcomp = diskcomp.cli:main`).
- **D-05:** Optional dependencies declared as extras: `diskcomp[rich]` installs `rich`; `smartmontools` is noted as a system package (not a pip dep) in README only.
- **D-06:** No PyPI publish this phase — `pyproject.toml` and build machinery are set up and verified locally, but the actual `pip publish` step is deferred until the user decides to release.

### CI (GitHub Actions)
- **D-07:** Test matrix: macOS-latest, ubuntu-latest, windows-latest × Python 3.8, 3.10, 3.12 (9 combinations). Runs `python -m unittest discover tests/`.
- **D-08:** No auto-publish step in CI. CI validates only — publish is a manual action.
- **D-09:** Single workflow file: `.github/workflows/ci.yml`. Trigger: push to main + pull_request.

### README (DIST-04)
- **D-10:** Practical + badges format: CI status badge + PyPI version badge at top. Sections: Install, Quick Start, Usage, Flags reference, Safety model, Optional enhancements (rich, smartmontools).
- **D-11:** No terminal recordings (asciinema) — text code blocks only. Maintainable and doesn't require re-recording when UI changes.
- **D-12:** README covers both install paths: `pip install diskcomp` and single-file download.

### Claude's Discretion
- Module concatenation order in `build_single.py` — Claude decides based on import dependency graph
- Badge URLs — Claude picks appropriate shields.io patterns for CI and PyPI
- Python version range in pyproject.toml classifiers — Claude fills based on tested matrix
- Whether to include a `Makefile` for `make build` convenience — Claude decides

</decisions>
