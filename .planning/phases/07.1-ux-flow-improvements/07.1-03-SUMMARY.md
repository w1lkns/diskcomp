## Accomplishments

### 1. extract_duplicate_files() (reporter.py)
- Extracts all unique file paths from classification results dict
- Flattens duplicates, unique_in_keep, unique_in_other categories
- Returns sorted list of paths for UI display
- Handles None and empty input gracefully
- Used as data source for file flagging UI

### 2. display_file_list() (RichProgressUI & ANSIProgressUI)
- RichProgressUI: Uses Rich Table with bold indices and dim styling for flagged
- ANSIProgressUI: Uses ANSI codes (BOLD, CYAN) with identical formatting
- Displays numbered list: "  1) /path/to/file.txt (5.2 MB)"
- Shows "[flagged]" marker for files in flagged_set
- Both classes have identical signatures and behavior
- Includes section header "Files to protect (flag these before deletion)"
- Includes blank lines before/after list

### 3. show_file_flagging() (cli.py)
- Main entry point for per-file flagging workflow
- Extracts all files from context.scan_results using extract_duplicate_files()
- Creates FileRecord objects with actual disk sizes
- Displays numbered file list with ui.display_file_list()
- Accepts range notation input: "1,3,5" or "1-3" or mixed "1,3-5"
- Handles 'b' key for back navigation (returns unchanged)
- Validates input with parse_selection_input()
- Stores selected paths in context.flagged_files (set, union with existing)
- Displays confirmation: "Flagged N file(s). These will be excluded from deletion."
- Graceful error handling: loops on invalid input via ui.error()

### 4. filter_candidates_by_flags() (deletion.py)
- Removes flagged files from deletion candidates before deletion
- Works with classification dict from DuplicateClassifier.classify()
- Filters all three categories: duplicates, unique_in_keep, unique_in_other
- Removes entire entries if ANY path is flagged (both keep and other)
- Returns new filtered dict with same structure
- Handles empty flagged_files set gracefully (returns unchanged)
- Used to prevent deletion of protected files during orchestration

### 5. Unit Tests (test_cli.py + test_deletion.py)
- 5 tests in test_cli.py:
  - test_extract_duplicate_files_flattens_all_files: verify extraction and sorting
  - test_extract_duplicate_files_handles_empty_results: verify empty handling
  - test_extract_duplicate_files_handles_none_input: verify None handling
  - test_show_file_flagging_displays_list_and_accepts_input: verify UI and parsing
  - test_show_file_flagging_returns_context_unchanged_on_back: verify back navigation

## Decisions Made

- **File extraction:** extract_duplicate_files() works directly with classification dict rather than creating FileRecord objects, since the dict is the actual runtime structure
- **Size calculation:** show_file_flagging() fetches actual file sizes from disk on-the-fly when creating FileRecord objects for display
- **Size format:** "X.X MB" with 1 decimal place precision for readability
- **UI polymorphism:** display_file_list() implemented identically in both RichProgressUI and ANSIProgressUI using identical numbering and formatting
- **Filtering approach:** filter_candidates_by_flags() creates new dict instead of mutating input (functional programming pattern)
- **Flag persistence:** context.flagged_files stored as set() for O(1) membership checking during filtering
- **Input validation:** show_file_flagging() reuses parse_selection_input() from Wave 2 for consistent range notation support

## Flow Diagram

```
run_post_scan_menu()
    └─→ User selects option 2 (flag files)
        └─→ show_file_flagging(context, ui)
            ├─→ extract_duplicate_files(context.scan_results) → [/path/a, /path/b, ...]
            ├─→ Create FileRecord objects with actual disk sizes
            ├─→ ui.display_file_list(files, context.flagged_files)
            └─→ Loop:
                ├─→ user input: "1,3,5" or "1-3" or "b"
                ├─→ parse_selection_input(input) → [1, 3, 5] (validated)
                ├─→ Convert to paths: {/path/a, /path/c, /path/e}
                ├─→ Union with context.flagged_files (add to existing)
                ├─→ ui.ok("Flagged N files. These will be excluded from deletion.")
                └─→ Return context with updated flagged_files

Later in deletion workflow:
    └─→ filter_candidates_by_flags(classification, context.flagged_files)
        └─→ Remove all entries containing flagged paths
        └─→ Return filtered classification dict
        └─→ Pass filtered results to DeletionOrchestrator
```
