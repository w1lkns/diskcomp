## Integration Workflow Diagram

```
main() [two-drive mode]
    ├─→ Scan both drives
    ├─→ Hash files
    ├─→ Classify (find duplicates)
    ├─→ Report write
    ├─→ Summary display
    ├─→ NavigationContext(scan_results, keep_path, other_path, report_path)
    ├─→ run_post_scan_menu(context, ui)
    │   ├─→ Loop: display 6-option menu
    │   ├─→ Option 1: show_folder_selection() → skip folders
    │   ├─→ Option 2: show_file_flagging() → flag files
    │   ├─→ Option 3: show_deletion_preview() → preview
    │   ├─→ Option 4: orchestrate_deletion(mode='interactive')
    │   │   ├─→ filter_candidates_by_flags(flagged_files)
    │   │   ├─→ DeletionOrchestrator.interactive_mode()
    │   │   └─→ return outcome
    │   ├─→ Option 5: orchestrate_deletion(mode='batch')
    │   │   ├─→ filter_candidates_by_flags(flagged_files)
    │   │   ├─→ DeletionOrchestrator.batch_mode()
    │   │   │   ├─→ Dry-run preview
    │   │   │   ├─→ Confirm: 'DELETE', 'b', or Ctrl+C
    │   │   │   ├─→ 'b' returns aborted=True
    │   │   │   └─→ 'DELETE' executes deletion
    │   │   └─→ return outcome
    │   └─→ Option 6: exit (no deletion)
    │       └─→ return 'skipped_deletion'
    │
    ├─→ Handle outcome
    │   ├─→ 'deleted': print success, return 0
    │   ├─→ 'aborted': print abort, return 0
    │   └─→ 'skipped_deletion': print no deletion, return 0
    │
    └─→ Close UI, return 0

State Preservation via NavigationContext:
    - selected_folders_skip: Set of folder paths to skip
    - flagged_files: Set of file paths to protect
    - scan_results: Original classification dict
    - keep_path, other_path, report_path: Drive info
    - Preserved across all navigation (back/forward)
```
