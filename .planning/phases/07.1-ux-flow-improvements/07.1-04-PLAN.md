  <action>
Add run_post_scan_menu(context: NavigationContext, ui: UIHandler) -> str function to diskcomp/cli.py. This is the central hub for post-scan navigation, called after scan and summary display complete.

Menu structure (show to user):
```
Next Steps:
  1) Skip certain folders (exclude before deletion)
  2) Flag individual files (mark 'do not delete')
  3) Review deletion preview
  4) Start deletion (Mode A: interactive)
  5) Start deletion (Mode B: batch)
  6) Exit without deletion
```

Behavior (while loop, never re-scans):
1. Display menu header "Next Steps"
2. Loop until user makes a valid choice (1-6):
   a. Use prompt_confirm("Enter your choice (1-6): ", ['1','2','3','4','5','6'], ui)
   b. Route based on choice:
      - '1': Call show_folder_selection(context, ui), update context, loop back to menu
      - '2': Call show_file_flagging(context, ui), update context, loop back to menu
      - '3': Call show_deletion_preview(context, ui) — preview without executing [see note below]
      - '4': Call orchestrate_deletion(context, ui, mode='interactive'), return result
      - '5': Call orchestrate_deletion(context, ui, mode='batch'), return result
      - '6': Return 'skipped_deletion' (user chose not to delete)

Return value: String indicating outcome ('deleted', 'aborted', 'skipped_deletion')

Implementation notes:
- use show_folder_selection() and show_file_flagging() from previous waves
- show_deletion_preview() can be a simple display (show which files will be deleted, respecting flagged_files)
- orchestrate_deletion() is a new wrapper (Task 2) that wraps DeletionOrchestrator with filtering

Docstring: Explain menu options, navigation flow, return values.
