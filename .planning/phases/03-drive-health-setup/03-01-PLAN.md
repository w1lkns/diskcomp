**Function 1: get_filesystem_type(mount_point: str) -> str**
- Input: path to a mounted drive (e.g., '/Volumes/MyDrive')
- Returns: filesystem type string (e.g., 'NTFS', 'HFS+', 'ext4')
- Logic:
  - If PSUTIL_AVAILABLE: use `psutil.disk_partitions()`, find matching mountpoint, return fstype
  - Else: call platform-specific subprocess function (see below), return result
  - On error: return 'UNKNOWN'

**Function 2: _get_filesystem_type_macos(mount_point: str) -> str**
- Platform: macOS only
- Uses subprocess to call `diskutil info <mount_point> | grep "Type:"`
- Parses output to extract filesystem type (e.g., "APFS", "NTFS")
- Returns type or "UNKNOWN" on error

**Function 3: _get_filesystem_type_linux(mount_point: str) -> str**
- Platform: Linux only
- Uses subprocess to call `df --output=fstype <mount_point>`
- Parses output line (skipping header) to extract fstype
- Returns type or "UNKNOWN" on error

**Function 4: _get_filesystem_type_windows(mount_point: str) -> str**
- Platform: Windows only
- Uses subprocess to call `wmic logicaldisk where name="{drive}:" get filesystem` (where drive is C, D, etc.)
- Parses CSV output to extract filesystem
- Returns type or "UNKNOWN" on error

**Function 5: check_drive_health(mount_point: str) -> HealthCheckResult**
- Input: path to mounted drive
- Returns: HealthCheckResult with space usage, filesystem, read-only status, warnings
- Logic:
  - Call `shutil.disk_usage(mount_point)` to get total/used/free bytes
  - Convert bytes to GB (divide by 1024^3), round to 2 decimals
  - Call `get_filesystem_type(mount_point)` for fstype
  - Call `os.access(mount_point, os.W_OK)` for write permission
  - Build warnings list:
    - If not writable: add "Drive is read-only: cannot write to this path"
    - If fstype == 'NTFS' and sys.platform == 'darwin': add "NTFS on macOS may be read-only without macFUSE+NTFS-3G write support"
  - Construct and return HealthCheckResult with all fields populated, smart_data=None (SMART is separate task)

**Function 6: get_smart_data(device_path: str) -> Optional[dict]**
- Input: device identifier (e.g., '/dev/sda', 'pd0')
- Returns: dict with {device, health_status, temperature_c, error} or None if smartctl unavailable
- Logic:
  - Try subprocess.run(['smartctl', '-j', device_path], capture_output=True, text=True, timeout=10)
  - If returncode in [0, 2] (0=ok, 2=warnings OK):
    - Parse JSON: data = json.loads(result.stdout)
    - Extract health_status from data['smart_status']['passed'] (convert bool to "PASSED"/"FAILED")
    - Extract temperature_c from data['temperature']['current_celsius']
    - Return dict with device, health_status, temperature_c, error=None
  - On FileNotFoundError: return None (smartctl not on PATH)
  - On subprocess.TimeoutExpired: return None (timeout)
  - On json.JSONDecodeError: return None (invalid JSON)
  - On Exception: return None (unexpected error, graceful skip)

**Pattern adherence:**
- Follow same style as scanner.py (function docstrings with Args/Returns)
- Use try/except for optional dependencies (psutil, smartctl)
- Error handling: catch and return defaults, never crash
- Cross-platform: detect sys.platform and call appropriate subprocess functions

**Function 1: benchmark_read_speed(mount_point: str, test_size_mb: int = 128, chunk_size_kb: int = 512) -> BenchmarkResult**
- Input:
  - mount_point: path to drive to test
  - test_size_mb: size of test file (default 128)
  - chunk_size_kb: read chunk size (default 512)
- Returns: BenchmarkResult with speed_mbps, duration_secs, bytes_read, success, error
- Logic:
  - Calculate test_size_bytes = test_size_mb * (1024 ** 2)
  - Calculate chunk_size_bytes = chunk_size_kb * 1024
  - Create temp file: `fd, temp_path = tempfile.mkstemp(dir=mount_point, prefix='diskcomp_bench_')`
  - Try/finally block:
    - Try:
      - Write test data: open temp file for 'wb', write test_size_bytes of b'\0' in chunks
      - Sleep 0.1 seconds (allow disk cache to flush)
      - Measure read speed:
        - start_time = time.time()
        - Open temp file for 'rb', read in chunks until EOF
        - Count bytes_read
        - elapsed = time.time() - start_time
      - Calculate speed_mbps = (bytes_read / (1024**2)) / elapsed (if elapsed > 0, else 0)
      - Return BenchmarkResult(mountpoint, speed_mbps, elapsed, bytes_read, success=True, error=None)
    - On Exception as e:
      - Return BenchmarkResult(mountpoint, 0, 0, 0, success=False, error=str(e))
    - Finally:
      - Try to delete temp_path with os.remove(), silently catch any errors

**Module docstring** (3-5 lines): Explain this module benchmarks sequential read speed on a drive.

**Pattern adherence:**
- Function has complete docstring with Args/Returns
- Error handling: catch all exceptions, return failed BenchmarkResult instead of crashing
- Timing: use time.time() for elapsed measurement
- Cleanup: always delete temp file in finally block
