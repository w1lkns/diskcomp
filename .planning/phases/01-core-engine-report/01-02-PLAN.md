Create diskcomp/hasher.py with a FileHasher class that implements SHA256 hashing with chunked reading for memory efficiency and error handling.

1. **FileHasher class** with methods:
   - `__init__(self, chunk_size: int = 8192)` — initialize with chunk size (default 8KB) for streaming reads
   - `hash_file(self, file_path: str) -> str` — compute SHA256 hash of a file:
     * Open file in binary mode ('rb')
     * Read in chunks of chunk_size
     * Feed each chunk to hashlib.sha256()
     * Return hex digest (lowercase)
     * On PermissionError: raise FileNotReadableError with reason "Permission denied"
     * On OSError: raise FileNotReadableError with reason "Cannot read file"
     * On IOError: raise FileNotReadableError with reason "IO error"
   - `hash_file_record(self, record: FileRecord) -> FileRecord` — hash a FileRecord and return updated copy:
     * Call hash_file(record.path)
     * Update record.hash with result
     * Catch FileNotReadableError, set record.error to error message
     * Return modified record

2. **Imports**:
   - import hashlib
   - import os
   - from diskcomp.types import FileRecord, FileNotReadableError

3. **Error handling**:
   - Catch PermissionError, OSError, IOError during file read
   - Raise FileNotReadableError with descriptive reason
   - Never crash — allow caller to decide how to handle

Example usage (comment in file):
```python
hasher = FileHasher(chunk_size=8192)
hash_value = hasher.hash_file("/path/to/file.txt")  # returns "abc123def456..."
```

4. **Performance note**: Chunked reading allows hashing large files (GB+) without loading entire file into memory. Use 8KB chunks as default (balance between memory and I/O).

Create diskcomp/reporter.py with DuplicateClassifier and ReportWriter classes for classifying files and generating CSV/JSON reports with atomic writes.

1. **DuplicateClassifier class** — classifies files as DUPLICATE or UNIQUE:
   - `__init__(self, keep_result: ScanResult, other_result: ScanResult)` — initialize with two ScanResult objects from scanner
   - `classify(self) -> dict` — classify all files and return dict:
     ```python
     {
         'duplicates': [
             {
                 'action': 'DELETE_FROM_OTHER',
                 'keep_path': '/Volumes/Keep/file.txt',
                 'other_path': '/Volumes/Other/file.txt',
                 'size_mb': 1.5,
                 'hash': 'abc123...',
             },
             ...
         ],
         'unique_in_keep': [...],  # files in keep but not in other
         'unique_in_other': [...],  # files in other but not in keep
         'summary': {
             'duplicate_count': N,
             'duplicate_size_mb': X,
             'unique_in_keep_count': M,
             'unique_in_keep_size_mb': Y,
             'unique_in_other_count': P,
             'unique_in_other_size_mb': Z,
         }
     }
     ```
   - Algorithm:
     * Build hash → file mapping for keep_result.files
     * For each file in other_result.files:
       - If hash exists in keep mapping → DUPLICATE → action='DELETE_FROM_OTHER'
       - Else → UNIQUE_IN_OTHER
     * For each file in keep_result.files:
       - If hash not in other mapping → UNIQUE_IN_KEEP

2. **ReportWriter class** — writes CSV and JSON reports atomically:
   - `__init__(self, output_dir: str = None, output_path: str = None)` — initialize with output directory or exact path:
     * If output_path provided, use it as-is
     * If output_dir provided, generate filename: `diskcomp-report-{YYYYMMDD}-{HHMMSS}.csv` (timestamp using datetime.datetime.now())
     * Default output_dir: home directory (os.path.expanduser("~"))
   - `write_csv(self, classification: dict, path: str = None)` — write CSV report atomically:
     * Use provided path, or self.output_path with .csv extension
     * Write headers: action, keep_path, other_path, size_mb, hash
     * Write duplicate rows, then unique rows
     * Use temp file → rename pattern for atomic writes:
       - Write to `{path}.tmp` first
       - On success, os.rename() to `{path}` (atomic on most OSes)
       - On error, clean up temp file
     * Raise exception if write fails
   - `write_json(self, classification: dict, path: str = None)` — write JSON report atomically:
     * Same atomic pattern as CSV
     * Write classification dict as-is to JSON
     * Pretty-print with indent=2 for readability

3. **Imports**:
   - import csv
   - import json
   - import os
   - import tempfile
   - from datetime import datetime
   - from diskcomp.types import FileRecord, ScanResult

4. **Example usage** (comment in file):
```python
classifier = DuplicateClassifier(keep_result, other_result)
classification = classifier.classify()
writer = ReportWriter(output_dir="/tmp")
writer.write_csv(classification)
```

5. **File size conversion**: When writing reports, convert size_bytes to size_mb using: `size_mb = round(size_bytes / (1024 ** 2), 2)`
